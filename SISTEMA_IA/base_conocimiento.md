# Base de conocimiento IA

## Objetivo

Documentar cómo los asistentes futuros podrán apoyarse en conocimiento canónico de PanaderIA™.

## Fuentes internas candidatas

- Sistema Maestro de Conocimiento.
- Sistema Maestro de Contenido.
- Sistema del Libro.
- Sistema de Curso.
- Sistema Boly™.
- Sistema de Calidad.
- Mapa Canónico del Curso.
- Sistemas multimedia, visuales y descargables.

## Principios

- La base de conocimiento debe priorizar fuentes internas aprobadas.
- Toda respuesta crítica debe poder rastrearse a documentos canónicos.
- La ausencia de fuente debe activar cautela o escalamiento humano.
- El conocimiento obsoleto debe retirarse o marcarse como archivado.

## Estados de conocimiento

- `aprobado`.
- `en_revision`.
- `experimental`.
- `archivado`.
- `no_autorizado`.

## Restricción

Este documento no construye una base vectorial, índice técnico ni sistema de recuperación real.
