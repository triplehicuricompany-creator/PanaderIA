# Modelos de IA utilizados

## Política general

PanaderIA™ podrá usar distintos modelos de IA según la tarea, siempre que se documenten su propósito, alcance, riesgos y nivel de revisión humana requerido.

Este documento no fija proveedores obligatorios. Define criterios para seleccionar modelos de forma responsable.

## Criterios de selección

- Calidad de razonamiento para tareas editoriales y técnicas.
- Capacidad de seguir instrucciones del proyecto.
- Manejo de contexto documental amplio.
- Seguridad y privacidad de datos.
- Trazabilidad de resultados.
- Costo operativo.
- Disponibilidad y estabilidad.
- Capacidad multimodal cuando sea necesaria.

## Categorías de uso

### Modelos de texto

Uso previsto:

- Revisión editorial.
- Organización documental.
- Diagnósticos internos.
- Resúmenes y reportes.
- Preparación de prompts y matrices.

### Modelos multimodales

Uso previsto:

- Revisión de imágenes, diagramas o assets.
- Soporte a producción multimedia.
- Interpretación visual de materiales de trabajo.

### Modelos especializados

Uso previsto:

- Tareas concretas con alto volumen o bajo riesgo.
- Clasificación documental.
- Extracción estructurada.
- Validaciones repetitivas.

## Registro mínimo por modelo

Cada modelo usado en producción deberá documentar:

- Nombre del modelo o proveedor.
- Fecha de adopción.
- Tareas autorizadas.
- Tareas prohibidas.
- Datos permitidos.
- Requisitos de revisión humana.
- Criterios de reemplazo.

## Reglas de control

- No usar modelos no evaluados para decisiones críticas.
- No enviar información sensible sin autorización.
- No mezclar fuentes oficiales con información no verificada sin distinguirlas.
- No asumir exactitud técnica sin validación.
