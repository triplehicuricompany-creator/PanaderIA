# Arquitectura Documental de Comunidad

## Capas conceptuales

| Capa | Función | Estado |
| --- | --- | --- |
| Identidad comunitaria | Define propósito, tono y límites de participación. | Conceptual |
| Canales | Lista espacios candidatos para interacción futura. | No activado |
| Moderación | Ordena criterios de revisión, escalamiento y prevención. | No operativo |
| Eventos | Prepara encuentros educativos futuros. | No calendarizado |
| Seguridad | Protege privacidad, reputación y convivencia. | Pendiente de validación |

## Relación con otros sistemas

- `SISTEMA_CLIENTES`: experiencia y necesidades de alumnos.
- `SISTEMA_SOPORTE`: atención de incidencias, sin sustituir soporte real.
- `SISTEMA_MARKETING`: comunicación pública solo con aprobación futura.
- `SISTEMA_LEGAL`: revisión de privacidad, términos y responsabilidades.
- `SISTEMA_SEGURIDAD`: protección documental y prevención de abuso.

Esta arquitectura no crea espacios activos ni procedimientos obligatorios.
