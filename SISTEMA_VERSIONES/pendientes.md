# Pendientes del Sistema de Versiones

Lista controlada de tareas futuras. Estos pendientes no modifican contenido existente ni representan aprobación automática.

## Alta prioridad

- Definir inventario inicial de archivos versionables.
- Crear matriz de compatibilidad entre Libro Maestro, curso y recursos multimedia.
- Establecer responsable o criterio de aprobación para versiones finales.

## Prioridad media

- Incorporar registro específico para asistentes IA y Boly.
- Definir formato de archivo para respaldos de versiones finales.
- Documentar criterios de revisión para Release Candidates.

## Prioridad baja

- Añadir ejemplos adicionales de nomenclatura.
- Crear checklist de cierre para versiones finales.
- Evaluar un índice cronológico de ediciones históricas.
