# Sistema Maestro de Versiones de PanaderIA™

Este directorio establece el sistema oficial de control de versiones para PanaderIA™. Su función es registrar, clasificar y preservar la evolución del proyecto sin modificar capítulos existentes ni reescribir contenido histórico.

## Propósito

- Centralizar el versionado del Libro Maestro, curso, recursos multimedia y asistentes IA.
- Mantener historial auditable de cambios relevantes.
- Separar versiones Beta, Release Candidate y finales.
- Permitir evolución futura sin pérdida de trazabilidad.

## Componentes del sistema

| Archivo | Función |
|---|---|
| `manifest.json` | Metadatos del sistema de versiones. |
| `versiones_libro.md` | Registro de versiones del Libro Maestro y ediciones derivadas. |
| `versiones_curso.md` | Registro de versiones del curso y módulos formativos. |
| `versiones_multimedia.md` | Registro de recursos visuales, audio, video y descargables. |
| `historial_cambios.md` | Bitácora general de cambios importantes. |
| `roadmap.md` | Plan de evolución previsto por etapas. |
| `pendientes.md` | Lista controlada de tareas futuras. |
| `politica_versionado.md` | Reglas oficiales para nombrar y aprobar versiones. |

## Principio rector

Toda nueva versión debe documentarse antes de publicarse. Los contenidos existentes se preservan como referencia histórica y solo se registran cambios mediante este sistema.
