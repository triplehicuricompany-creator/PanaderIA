# ASSET_UPLOAD_MAP.md

## Propósito

Este documento explica exactamente dónde colocar cada archivo real de PanaderIA™ dentro de la estructura oficial `public/assets/`.

Debe usarse antes de subir cualquier activo nuevo para evitar duplicados, nombres incorrectos o archivos colocados en carpetas equivocadas.

No se deben agregar imágenes falsas, archivos temporales o contenido sin aprobación. Este mapa está diseñado únicamente para la carga de activos reales.

---

# Videos

## Ubicación para referencias locales

`public/assets/videos/curso-bolillo/`

## Formatos recomendados para archivos locales

- MP4 H.264.
- WEBM optimizado.
- Poster en JPG o WEBP.

## Nombre sugerido

`curso-bolillo-mod01-leccion01-v001.mp4`

## Cómo registrar enlaces externos de YouTube, Vimeo o almacenamiento privado

Los videos externos no deben duplicarse como archivos dentro del repositorio si estarán alojados en una plataforma externa. Deben registrarse en una fuente de contenido o configuración futura con estos datos mínimos:

- `id`: identificador interno de la lección.
- `provider`: `youtube`, `vimeo`, `private-storage` o `streaming-secure`.
- `url`: enlace público, privado o firmado según corresponda.
- `thumbnail`: ruta al poster o imagen miniatura.
- `module`: módulo del curso.
- `lesson`: lección o capítulo.
- `accessLevel`: `standard`, `premium` o `admin`.
- `qrCode`: código QR asociado si aplica.
- `version`: versión del video.

## Ejemplo documental de registro futuro

```text
id: curso-bolillo-mod01-leccion01
provider: vimeo
url: https://vimeo.com/ejemplo-privado
thumbnail: public/assets/videos/curso-bolillo/curso-bolillo-mod01-leccion01-poster-v001.webp
module: modulo-01-introduccion
lesson: leccion-01-bienvenida
accessLevel: premium
qrCode: QR-BOLILLO-M01-L01
version: v001
```

## Nota de producción

Para una plataforma comercial, los videos privados deben vivir en almacenamiento/streaming seguro con control de acceso. `public/assets/videos/` puede servir para assets públicos, posters, referencias locales o demos aprobadas.

# Reglas generales

- No usar espacios en nombres de archivo.
- Usar minúsculas.
- Usar guiones medios.
- Mantener versión oficial marcada como `-oficial`.
- No borrar archivos antiguos sin respaldo.
- Documentar cada cambio de activo.
- No subir archivos temporales como `final-final.png`, `nuevo.png`, `prueba.jpg` o `sin-nombre.png`.
- No mezclar assets de marca con assets de curso.
- No colocar videos privados definitivos en rutas públicas sin revisión de seguridad.
- No reemplazar activos oficiales sin incrementar versión o conservar historial.

## Flujo recomendado para cargar un activo real

1. Identificar el tipo de activo.
2. Buscar su ubicación en este mapa.
3. Revisar el `README.md` de la carpeta destino.
4. Nombrar el archivo en minúsculas, con guiones medios y versión.
5. Verificar que no exista un duplicado.
6. Colocar el archivo en la carpeta correcta.
7. Documentar el cambio en el control interno correspondiente.
8. Actualizar referencias en código o contenido solo cuando el activo esté aprobado.

## Relación con ACTIVOS_OFICIALES.md

`ACTIVOS_OFICIALES.md` define la estructura general de activos y los criterios de versión oficial. Este archivo, `ASSET_UPLOAD_MAP.md`, funciona como mapa operativo para saber exactamente dónde subir cada archivo real.
