# Convención de versiones y ediciones

## Objetivo

Establecer una convención única para identificar versiones digitales, impresas y comerciales del Libro Maestro Premium PanaderIA™.

## Formato recomendado

`LMP-PANADERIA-[EDICION]-vMAJOR.MINOR.PATCH-[FORMATO]-[AAAA-MM-DD]`

Ejemplos:

- `LMP-PANADERIA-ED01-v1.0.0-PDF-2026-07-06`
- `LMP-PANADERIA-ED01-v1.0.1-EPUB-2026-07-06`
- `LMP-PANADERIA-ED02-v2.0.0-IMPRESO-2026-07-06`

## Criterios SemVer editorial

| Cambio | Uso |
|---|---|
| MAJOR | Nueva edición, reestructura profunda, nuevos capítulos o cambios pedagógicos mayores. |
| MINOR | Nuevas secciones, recursos, anexos, mejoras visuales o ampliaciones sin romper continuidad. |
| PATCH | Correcciones de erratas, enlaces, formato, metadatos o ajustes menores. |

## Estados de versión

- `borrador`: trabajo interno no exportable.
- `candidato`: listo para revisión final.
- `aprobado`: autorizado para exportación.
- `publicado`: disponible comercialmente.
- `retirado`: ya no debe venderse.
- `reemplazado`: sustituido por una versión posterior.

## Estrategia de actualización

1. Registrar cada cambio con fecha, responsable, motivo e impacto.
2. Mantener compatibilidad entre edición digital e impresa cuando sea posible.
3. Publicar parches solo para errores concretos.
4. Reservar nuevas ediciones para mejoras sustanciales.
5. Notificar a compradores cuando la actualización afecte recursos prometidos.
6. Conservar historial de versiones retiradas para soporte y auditoría.
