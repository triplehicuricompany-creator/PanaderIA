# Soporte

## Objetivo

Establecer el modelo documental de soporte para consultas, incidencias y escalamiento.

## Alcance documental

- Categorías de solicitud.
- Prioridades y tiempos conceptuales.
- Responsables y rutas de escalamiento.
- Registro y cierre de casos.
- Retroalimentación hacia calidad y producto.

## Reglas de control

1. Este documento define infraestructura, no ejecución comercial.
2. Toda iniciativa derivada debe registrar responsable, estado, fecha y criterio de revisión.
3. Ningún contenido aquí constituye página de venta, campaña, copy comercial ni autorización de publicación.
4. Los cambios que afecten productos, clientes o soporte requieren revisión humana.
5. Se debe mantener trazabilidad con el manifiesto del sistema y con los sistemas relacionados.

## Estados permitidos

- propuesto
- borrador
- en_revision
- aprobado
- activo
- pausado
- retirado

## Restricciones

No crear páginas de venta, textos comerciales, campañas, código ni modificaciones a capítulos existentes.
