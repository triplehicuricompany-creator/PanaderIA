# Arquitectura del Sistema Maestro de Ventas PanaderIA™

## Objetivo

Definir la estructura documental que permite planear, revisar y gobernar ventas sin ejecutar campañas ni producir piezas comerciales finales.

## Alcance documental

- Gobierno documental y manifiesto oficial.
- Oferta estructural: productos, libro, cursos, membresías, licencias y paquetes.
- Operación comercial: embudo, CRM conceptual, seguimiento y conversión.
- Relación continua: postventa, fidelización y soporte.
- Evolución: roadmap, métricas futuras y aprendizaje documentado.

## Reglas de control

1. Este documento define infraestructura, no ejecución comercial.
2. Toda iniciativa derivada debe registrar responsable, estado, fecha y criterio de revisión.
3. Ningún contenido aquí constituye página de venta, campaña, copy comercial ni autorización de publicación.
4. Los cambios que afecten productos, clientes o soporte requieren revisión humana.
5. Se debe mantener trazabilidad con el manifiesto del sistema y con los sistemas relacionados.

## Estados permitidos

- propuesto
- borrador
- en_revision
- aprobado
- activo
- pausado
- retirado

## Restricciones

No crear páginas de venta, textos comerciales, campañas, código ni modificaciones a capítulos existentes.
