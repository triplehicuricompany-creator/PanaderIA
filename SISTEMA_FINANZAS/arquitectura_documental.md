# Arquitectura Documental Financiera

## Capas del sistema

| Capa | Función | Estado |
|---|---|---|
| Gobernanza | Define quién revisa, aprueba y escala decisiones financieras. | Conceptual |
| Planeación | Organiza presupuestos, costos, gastos, ingresos y márgenes futuros. | Conceptual |
| Control | Define reportes, flujo de caja, riesgos y controles internos. | Conceptual |
| Cumplimiento | Documenta necesidades fiscales y contables pendientes. | Pendiente de asesoría |
| Evolución | Ordena mejoras, integraciones y futuras herramientas. | Roadmap |

## Relación con otros sistemas

- `SISTEMA_COMERCIAL`: aporta contexto de productos, ofertas y estrategia comercial sin fijar precios reales.
- `SISTEMA_VENTAS`: aporta categorías de venta futuras sin registrar transacciones.
- `SISTEMA_CLIENTES`: aporta segmentos conceptuales sin usar datos personales reales.
- `SISTEMA_LEGAL`: aporta revisión de obligaciones, contratos y permisos.
- `SISTEMA_INTEGRACIONES`: documentará futuras conexiones solo si una misión posterior lo autoriza.

## Exclusión técnica

Esta arquitectura no define software contable, bancos conectados, pasarelas de pago, facturación, impuestos calculados ni tableros financieros funcionales.
