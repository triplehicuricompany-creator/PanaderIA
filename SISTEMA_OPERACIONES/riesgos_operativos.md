# Riesgos operativos conceptuales

| Riesgo | Descripción | Mitigación documental |
| --- | --- | --- |
| Duplicidad de sistemas | Dos áreas documentan la misma función. | Revisar arquitectura antes de crear flujos. |
| Ejecución prematura | Un documento conceptual se usa como procedimiento activo. | Marcar restricciones y estado no ejecutable. |
| Falta de responsable funcional | Una tarea futura no tiene sistema dueño. | Asignar rol funcional, no persona real. |
| Contradicción documental | Una nueva regla contradice una misión cerrada. | Escalar a gobernanza. |
| Dependencia técnica no validada | Se asume una herramienta inexistente. | Coordinar con integraciones y seguridad. |
| Métricas sin evidencia | Se reportan números no medidos. | Mantener indicadores como conceptuales. |
