# Calidad de Datos

## Propósito

Definir criterios conceptuales para evaluar la calidad de datos en fases futuras, antes de cualquier implementación técnica.

## Dimensiones de calidad

- Completitud.
- Exactitud.
- Consistencia.
- Oportunidad.
- Unicidad.
- Validez.
- Trazabilidad.
- Pertinencia.

## Controles conceptuales

- Diccionario de datos aprobado.
- Reglas de validación documentadas.
- Propietarios por dominio.
- Revisión de sesgos y anomalías.
- Registro de cambios.
- Retención y eliminación conceptual.

## Restricción

Este documento no crea datasets, esquemas físicos, pipelines ni reglas ejecutables.
