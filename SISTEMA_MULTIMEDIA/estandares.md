# Estándares multimedia oficiales

## Estándares generales

- Todo recurso debe tener propósito definido.
- Todo archivo debe cumplir la nomenclatura oficial.
- Todo recurso debe pasar control de calidad antes de integrarse.
- Todo recurso debe tener derechos de uso claros.
- Todo recurso debe conservar versión master cuando aplique.
- Todo recurso debe estar asociado a un capítulo, módulo, área o uso comercial.

## Formatos permitidos por categoría

| Categoría | Maestros | Entrega | Web |
| --- | --- | --- | --- |
| Fotografía | RAW, TIFF, PSD | TIFF, PNG, JPG | WebP, JPG |
| Ilustración | AI, SVG, PSD | PDF, PNG, SVG | SVG, WebP, PNG |
| Video | MOV, ProRes, proyecto editable | MP4, MOV | MP4, WebM |
| QR | SVG, PDF | SVG, PNG, PDF | SVG, PNG |
| Audio | WAV, proyecto editable | WAV, FLAC | MP3, AAC, M4A |
| Animación | AE, Blender, SVG, Lottie fuente | MP4, WebM, GIF | WebM, Lottie, WebP |

## Resoluciones mínimas recomendadas

- Imagen editorial impresa: 300 ppp al tamaño final.
- Imagen digital general: 1600 px lado largo mínimo.
- Miniatura de video: 1280 x 720 px.
- Video horizontal: 1920 x 1080 px mínimo.
- Video vertical: 1080 x 1920 px mínimo.
- QR para impresión: SVG preferente o PNG mínimo 1000 x 1000 px.
- Audio master: WAV 48 kHz / 24 bit.

## Control de versiones

- `v01`: primera versión funcional.
- `v02`, `v03`: cambios de contenido o edición.
- `final`: versión lista para revisión final.
- `aprobado`: versión validada para integración.
- `reemplazado`: recurso obsoleto conservado por historial.

## Criterios de rechazo

Un recurso debe rechazarse si presenta:

- Baja resolución para su destino.
- Nombre fuera de convención.
- Derechos de uso no claros.
- Mala iluminación, audio ininteligible o compresión excesiva.
- Información visual incorrecta o contradictoria.
- QR que no lee correctamente.
- Archivo corrupto, incompleto o sin destino definido.
